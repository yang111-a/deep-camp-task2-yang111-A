## AI 处理流程分析

### Phase 1：阅读理解与词汇筛选

输入：原始英文段落（约 120 单词）
处理：
1. 语义理解：识别文章主题（digital nomad 现象）、论证结构（利弊分析）
2. 词汇分层：按 CET-4 标准剔除基础词（and, but, is）和过偏词（无），保留中高频学习价值词
3. 表格生成：按指定格式输出 19 个词

### Phase 2：记忆卡深度生成

输入：4 个用户指定词汇
处理：
1. 词根分析：查证词源（希腊/拉丁词根），确保准确性
2. 联想技巧：创造中文谐音或形象联想
3. 搭配提取：基于语料库高频共现给出自然搭配
4. 难度评级：根据四级考频和用法复杂度判定

### Phase 3：格式化与项目化

输入：所有中间产物（.txt 文件、对话记录）
处理：
1. Markdown 转写：表格、标题、代码块
2. 补充内容：使用说明、复习计划
3. 目录设计：按场景-输出-参考三层组织

## 关键技术决策

| 决策点 | 选择 | 原因 |
|--------|------|------|
| GitHub 上传方式 | API (curl) 代替 git push | 网络直连不稳定，API 走 HTTPS 更可靠 |
| 难度评级 | 4 词全为 ⭐⭐ | 均为四级高频考点，不仅需认知更需会用 |
| 记忆技巧 | 不确定写"无" | 避免编造误导，宁缺毋滥 |
