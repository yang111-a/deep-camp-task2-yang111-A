# 用户原始提示词汇编

本文档记录了在本任务中用户对 AI 发出的所有提示词（Prompt），按对话顺序排列。

---

## Prompt 1：记忆查询

```
有昨天建仓库的记忆吗
```

**目的**：确认 AI 是否保留历史会话记忆。
**结果**：无相关记忆，触发后续"新建仓库"任务。

---

## Prompt 2：仓库创建

```
在GitHub新建仓库 deep-camp-task2-yang111-A，公开，先放一个README.md，里面暂时写一个标题即可
```

**目的**：创建任务用的公开 GitHub 仓库，写入初始 README。
**结果**：通过 GitHub API 创建仓库 + 上传 README.md。

---

## Prompt 3：阅读理解 + 词汇筛选

```
我下面给你一篇四级阅读真题原文。请你完成以下任务：
1. 先通读全文，理解大意。
2. 标注出其中【对四级考生来说】比较陌生、但高频出现的重要词汇。
3. 把这些词列成表格，格式为：【单词 | 词性 | 中文释义 | 文中原句】。

请注意：
- 不要列太简单的词（如 and、but、is、have）
- 也不要列太生僻的专业术语（除非它在文中反复出现）
- 控制在15-20个词左右

以下是原文：
[Digital Nomad Passage]
```

**目的**：以结构化方式提取四级核心词汇。
**结果**：19 词词汇表 + 全文大意总结。

---

## Prompt 4：保存为本地文件

```
帮我在桌面上创一个记事本并把你的这些词汇整理进去
```

**目的**：将词汇表保存为可查看的本地文件。
**结果**：`四级词汇整理_数字游民.txt` 写入桌面。

---

## Prompt 5：记忆卡深度生成

```
下面是我从四级真题中人工筛选出的生词列表，请你针对【每一个词】生成一份"记忆卡"，包含：

1. 单词 + 词性 + 中文释义
2. 真题原句（我已有的就保留，没有的请你补充一个四级难度例句）
3. 词根拆解或记忆技巧（帮助记忆，不会的写"无"）
4. 1-2个常见搭配/短语
5. 难度评级：⭐（认知即可）或 ⭐⭐（需掌握用法）

注意：记忆技巧如果没把握，请写"无"，不要编造。

我的词列表：
nomad | n. | 游牧者 | The concept of "digital nomad"...
remotely | adv. | 远程地 | ...use technology to work remotely...
flexibility | n. | 灵活性 | This lifestyle offers flexibility...
drawn to | phr. | 被吸引 | ...are drawn to this way of life...
```

**目的**：对重点词汇进行深度加工，生成可复用的记忆卡。
**结果**：`四级记忆卡_4词.txt`，含词根拆解、搭配、难度评级。

**Prompt 设计亮点**：
- 明确"不会写无，不编造"的边界约束，防 AI 幻觉
- 指定输出字段标准化，便于后续格式化

---

## Prompt 6：Markdown 规范化

```
请把我上面生成的"记忆卡"内容，整理成一份规范的Markdown文档，要求：

1. 标题：【四级真题生词记忆卡——Passage 1（数字游民篇）】
2. 开头写一段100字以内的"使用说明"
3. 每个词用二级标题 ### 单独列出
4. 内容用表格清晰呈现
5. 末尾加一段"下一步复习计划"（3-5条具体行动）
```

**目的**：将 .txt 记忆卡转为结构化 Markdown 文档。
**结果**：`四级真题生词记忆卡_Passage1_数字游民篇.md`。

---

```
Markdown保存为 outputs/vocabulary_card.md
```

**目的**：将文档按项目约定路径存档。

## Prompt 8：项目结构搭建 + 填充 + 提交

```
帮我在当前目录下建立一个完整的任务二项目结构，包含以下文件和文件夹：

1. README.md（根目录）
2. /task2-ai-scenario/ 文件夹，里面包含：
   - scenario_description.md
   - task_goal.md
   - ai_workflow.md
   - prompts.md
   - comparison.md
   - reflection.md
3. /screenshots/ 文件夹
4. /outputs/ 文件夹，里面放 vocabulary_card.md
5. /sources/ 文件夹，里面放 references.md

建好之后，把我刚才对话里生成的所有内容填入对应的文件中。
填完后帮我提交到GitHub。
```

**目的**：建立标准化项目结构，整合所有输出，推送到远程仓库。
**结果**：完整项目目录 + 所有文件内容填充 + GitHub 提交（待执行）。

---

## Prompt 设计总结

| 特征 | 说明 |
|------|------|
| 渐进式细化 | 从整体（全文理解）→ 局部（19 词）→ 焦点（4 词）→ 结构化（Markdown）|
| 边界约束 | 明确排除标准（太简单/太生僻）、防幻觉指令（"不会写无"）|
| 格式先行 | 每次交互预先指定输出格式，减少后续调整成本 |
| 多轮迭代 | 每次输出作为下一次输入的基础，逐步精炼 |
